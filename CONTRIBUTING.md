# Contributing

Thank you for considering contributing to CS Notes! Contributions help the repository grow and become more useful for everyone. Please follow these guidelines to make the process smooth.

How to contribute

- Issues: Open an issue to discuss new content ideas, corrections, or larger changes before implementing them.
- Pull requests: Fork the repository, create a topic branch, and open a pull request from that branch into the repository's `main` branch.

Content organization

- Place notes and resources in a clear, relevant folder (for example `algorithms/`, `data-structures/`, `systems/`).
- Use descriptive filenames and add a short header in each file describing its purpose.

Formatting

- Use Markdown for notes (`.md`). Keep formatting consistent and include code blocks where appropriate.
- If adding code, include language fences and short runnable examples when possible.

Pull request checklist

- [ ] The change is in the appropriate folder and follows naming conventions.
- [ ] Files include an explanatory header or front matter where relevant.
- [ ] Spelling and grammar have been checked.
- [ ] New content includes references or links for further reading when applicable.

License and attribution

- By contributing, you agree to license your contributions under the repository's license (see LICENSE file).
- If adding content copied from elsewhere, ensure it is permitted and include attribution and a link to the source.

Communication

- Mention maintainers by @username in issues or PRs if you need review or guidance.

Thank you!
